# Outlook 注册验证码解决方案

## 问题分析
Outlook 注册页面使用多种验证码技术，包括：
1. **reCAPTCHA v2/v3** - 谷歌验证码
2. **hCaptcha** - 替代验证码
3. **短信验证码** - 需要手机号
4. **邮箱验证码** - 需要已有邮箱

## 解决方案

### 方案1：使用已有账号（推荐）
避免验证码问题的最佳方案：

1. **手动注册 Outlook 账号**
   - 访问 https://signup.live.com
   - 手动完成注册流程
   - 记录邮箱和密码

2. **使用脚本的"已有账号"模式**
   - 运行 `使用已有账号.bat`
   - 选择"使用 Outlook 邮箱登录"
   - 输入手动注册的账号信息

### 方案2：使用 GitHub 账号
如果 Outlook 验证码太难，可以使用 GitHub 账号登录 Stackryze：

1. **确保有 GitHub 账号**
   - 访问 https://github.com 注册
   - 完成邮箱验证

2. **使用脚本的 GitHub 登录模式**
   - 运行 `使用已有账号.bat`
   - 选择"使用 GitHub 账号登录"

### 方案3：手动登录模式
完全手动操作，脚本只负责 Cloudflare 托管：

1. **手动注册 Stackryze 域名**
   - 访问 https://domain.stackryze.com
   - 手动登录并注册域名

2. **使用脚本的"手动登录"模式**
   - 运行 `使用已有账号.bat`
   - 选择"手动登录"
   - 输入已注册的域名

### 方案4：改进自动注册脚本
如果坚持使用自动注册，可以尝试以下改进：

#### 4.1 增加超时时间
修改 `Outlook自动注册.py` 中的超时设置：
```python
# 将 timeout=10000 改为 timeout=30000
await element.is_visible(timeout=30000)
```

#### 4.2 添加重试机制
```python
async def wait_and_click_with_retry(page, selectors, max_retries=3):
    for attempt in range(max_retries):
        for selector in selectors:
            try:
                element = page.locator(selector)
                if await element.is_visible(timeout=5000):
                    await element.click()
                    return True
            except:
                continue
        if attempt < max_retries - 1:
            await page.wait_for_timeout(2000)
    return False
```

3. 选择"使用 Outlook 邮箱登录"

## 故障排除

### 问题1：页面加载超时
- 检查网络连接
- 尝试使用 VPN
- 增加超时时间

### 问题2：验证码无法识别
- 使用手动注册方案
- 尝试不同的浏览器
- 清除浏览器缓存

### 问题3：脚本自动关闭
- 检查 Python 依赖
- 运行 `快速诊断.bat`
- 查看错误日志
